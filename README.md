# Spring-Boot3-Voll-Med-API<br>

## Desenvolvimento de uma API Rest
<p>- Desenvolver algumas funcionalidades: <b>CRUD(Create, Read, Update, Delete -> Cadastro, Listagem, Atualização e Exclusão)</b></p>
<p>- Validações das informações que chegam na API, utilizando o <b>Bin Validation</b></p>
<p>- Conceitos de <b>Paginação e Ordenação</b> das informações que a API vai devolver.</p>


## Tudo isso utilizando as tecnologias:
<p>- <b>Spring Boot 3</b>;</p>
<p>- <b>Java 17</b>;</p>
<p>- <b>Lombok</b> (Anotação para geração de códigos repetitivos, como getter e setter, toString, construtores. Tudo via anotação;</p>
<p>- <b>Banco de dados:</b> MySQL / Flyway (biblioteca que foi usada para ter o controle do histórico de evolução do meu banco de dados, conceito chamado de "Migration");</p>
<p>- <b>JPA / Hibernate:</b> a camada de persistência(JPA) da aplicação, foi feita utilizando <b>Hibernate</b> como implementação, e utilizando junto dos módulos do Spring Boot para tornar mais esse processo simples possível;</p>
<p>- <b>Maven:</b> Para gerencimaneto de dependências e como ferramenta para gerar o <b>build</b> da aplicação;</p>
<p>- Como o foco vai ser na API Rest(apenas no Back End), não vai ter interface gráfica, porém para <b>teste</b> da API utilizei o <b>INSOMNIA</b>;</p>




## QUAL VAI SER O PROJETO:
<p>Uma clinica médica fictícia;</p>
<p>Ela precisa de uma aplicativo para controlar o <b>CADASTRO DE MÉDICOS, DE CLIENTES, E AGENDAMENTO DE CONSULTAS</b>.</p>
